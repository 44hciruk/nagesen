## 📱 ページ構成

### 1. ロード画面（初回表示）
- ヨーヨーのアニメーション
- 「投げ銭を贈る準備中...」的なテキスト
- 1-2秒で自動的にメイン画面へ遷移

### 2. メイン画面
- **ヒーロー**: ヨーヨーイラスト（大きく、目立つ）
- **テキスト**: 「投げ銭を贈る」（大きく、太く、目立つ）
- **サブテキスト**: 「ヨーヨーをタッチして、投げ銭を贈りましょう！」
- **CTA**: 「PayPayを開く」ボタン（目立つ、大きい）
- **キャンセル**: 「戻る」ボタン（小さめ）

### 3. 背景エフェクト
- コインが落下するアニメーション
- 光のエフェクト・スパークル
- 全体的に祝い感・楽しさを演出

## 🎬 アニメーション仕様

### ヨーヨーのスピン
```
- 回転速度: 2秒で1回転
- 無限ループ
- 光のエフェクト付き
```

### コインの落下
```
- 複数のコインが上から下へ落下
- ランダムな開始時間
- 透明度のフェード
- 3-4秒のループ
```

### テキストの出現
```
- フェードイン: 0.5秒
- スライドイン: 上から下へ
- ディレイ: 段階的に出現
```

### ボタンのホバー
```
- スケール: 1.05倍
- 色の変更: より明るく
- シャドウの追加
- トランジション: 0.2秒
```

## 🎯 UX設計

### ユーザーフロー
1. NFC Toolsで設定したURLをタップ
2. ブラウザ確認ダイアログ「このURLを開きますか？」→ ユーザーが「開く」をタップ
3. 中間ページが開く
4. ロード画面表示（1-2秒）
5. メイン画面表示（「投げ銭を贈る」テキスト・ヨーヨーアニメーション）
6. ユーザーが「PayPayを開く」ボタンをタップ
7. PayPayのマイコード画面へリダイレクト

### ユーザーの心理
- **ロード画面**: 「何が起こるんだろう？」という期待感
- **メイン画面**: 「あ、投げ銭のジョークだ！」という気づき・笑顔
- **ボタンタップ**: 「PayPayを開こう」という自然な流れ

## 🛠️ 技術実装

### ファイル構成
```
client/src/
├── pages/
│   └── Redirect.tsx       # リダイレクトページ（メイン）
├── components/
│   ├── YoyoAnimation.tsx   # ヨーヨーアニメーション
│   └── CoinRain.tsx        # コイン落下アニメーション
└── hooks/
    └── useRedirect.ts      # リダイレクトロジック
```

### URLパラメータ
- `?target=paypay` - PayPayへリダイレクト
- 将来的に拡張可能（他の決済サービスなど）

### リダイレクト先
- PayPayのマイコードURL: `https://qr.paypay.ne.jp/p2p01_nd8UMxxYcZOhhHUu`
